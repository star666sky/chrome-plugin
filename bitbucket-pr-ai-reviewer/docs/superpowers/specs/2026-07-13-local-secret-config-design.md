# Local Secret Configuration Design

## Goal

Allow each user to keep Bitbucket and DeepSeek credentials in a local configuration file without placing real secrets in Git history.

## File layout

- Track `src/local-default-settings.example.js` as the shareable template.
- Keep `bitbucketToken` and `deepseekApiKey` as empty strings in the template.
- Ignore `src/local-default-settings.js` in Git. This is the real per-user configuration file.
- Keep the existing `settings.js` import contract unchanged so the runtime behavior remains synchronous.

## Setup flow

After cloning the repository, a user runs:

```bash
cp src/local-default-settings.example.js src/local-default-settings.js
```

The user then fills their own credentials in `src/local-default-settings.js` before loading the unpacked extension. Because the runtime uses a static import, the extension requires this copied file to exist.

## README changes

Translate the project README into Simplified Chinese, including installation, configuration, usage, security, implementation notes, and verification instructions. Keep commands, paths, API names, product names, and identifiers unchanged where translation would reduce accuracy.

The Chinese README must also:

1. Require copying the example file before loading the extension.
2. Identify the two credential fields that users must fill locally.
3. Explain that `src/local-default-settings.js` is ignored and must never be force-added to Git.
4. Warn users not to share or package an extension directory containing real credentials.
5. Preserve the existing option-page configuration guidance where applicable.

## Generated-file ignore rules

Replace the current imprecise ignore entries with explicit root-level rules that:

- Ignore `.idea/` and `*.iml` files generated by WebStorm and other JetBrains IDEs.
- Ignore macOS `.DS_Store` files.
- Ignore only `bitbucket-pr-ai-reviewer/src/local-default-settings.js` as the real local secret configuration.
- Keep `bitbucket-pr-ai-reviewer/src/local-default-settings.example.js` trackable.

Existing generated files remain untracked and are not committed.

## Migration

- Rename the currently tracked `src/local-default-settings.js` to `src/local-default-settings.example.js`.
- Create a local ignored copy at `src/local-default-settings.js` so the current checkout remains runnable.
- Do not put any real credential value into either committed files or implementation commands.

## Verification

- Confirm the example file is tracked and contains empty credential values.
- Confirm the real local file is ignored and absent from `git ls-files`.
- Confirm a normal `git add .` does not stage the local configuration file.
- Confirm `.idea/`, `*.iml`, and `.DS_Store` are ignored and remain untracked.
- Confirm the README content is in Simplified Chinese while commands and identifiers remain accurate.
- Run the existing focused test suite (`npm test`).
- Run `git diff --check` on the resulting changes.

## Scope

No asynchronous JSON loader, settings storage redesign, credential encryption, packaging automation, or unrelated refactoring is included.
